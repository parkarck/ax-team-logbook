RAG란 무엇인가?

RAG = 검색 증강 생성 (Retrieval-Augmented Generation)

똑똑한 로봇에게 도서관을 이용하게 해주는 것!

  • 검색 (Retrieval) = 정보 찾기
  • 증강 (Augmented) = 더하기
  • 생성 (Generation) = 새로 만들기

→ 더 나은 답변을 만들기 위해 정보를 찾는 것!

📖

도서관 비유

세상의 모든 책을 다 읽었지만 세부 정보를 잊어버린 친구...

RAG = 친구에게 도서관 이용증을 씌워주는 것!

  1. 도서관에서 빠르게 검색
  2. 중요한 부분을 읽기
  3. 정확한 답변 제공
📚 AI + 도서관 이용증 = 더 나은 답변! 🎯
🗂️

작동 원리 (1단계)

지식 저장소

수백만 권의 책이 있는 거대한 디지털 도서관!

  • 문서 - 책, 기사, 웹페이지
  • 청크 - 문서의 작은 조각들
  • 임베딩 - 의미를 위한 특별한 코드
  • 벡터 데이터베이스 - 코드를 저장하는 곳
🗄️ 서랍장 같아요!
❓

작동 원리 (2단계)

질문하기

질문을 해요!

"판다는 뭘 먹어?"

RAG 시스템은:

  1. 질문을 이해해요
  2. 특별한 코드(임베딩)로 변환해요
  3. 이 코드로 검색해요
👨‍🏫 사서에게 묻는 것과 같아요!
🔍

작동 원리 (3단계)

검색하기

시스템이 관련 정보를 검색해요!

이런 상황과 같아요:

  • 카탈로그에서 "판다 먹이" 검색
  • 판다와 대나무에 대한 책 찾기
  • 가장 관련성 높은 장 선택

→ 질문과 가장 비슷한 정보를 찾아요!

⚡ 검색 엔진 사용하기!
✨

작동 원리 (4단계)

증강 생성

AI가 정보를 사용해서 답변을 만들어요!

AI는 이제 다음을 가지고 있어요:

  • ✅ 당신의 원래 질문
  • ✅ 찾은 관련 정보

→ 이것들을 결합해서 도움이 되는 답변을 만들어요!

📝 참고 도서를 사용해서 보고서 쓰기!
💡

RAG가 왜 중요할까?

  1. ✓ 정확한 답변 - 실제 출처를 확인
  2. ✓ 최신 정보 - 재학습 없이 최신화
  3. ✓ 오류 감소 - 실수와 환각 감소
  4. ✓ 프라이버시 - 개인 문서를 안전하게 사용
  5. ✓ 설명 가능 - 출처를 표시
📖 RAG = 책을 보며 치는 시험!

실제 사례

🏢 고객 서비스

문제점: 지원 담당자가 수백 개의 정책을 기억해야 함

해결책: RAG가 회사 문서를 검색해서 정확한 정책을 찾음!

🏥 의료 연구

문제점: 매일 새로운 연구 발표, 의사가 모두 읽을 수 없음

해결책: RAG가 최신 저널을 검색해서 치료법을 요약!
